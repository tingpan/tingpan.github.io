使用Github和Hexo搭建个人博客1——配置本地Hexo环境

One Punch CAT
2015-01-31
hexo

Hexo是一个基于node.js的非常简单又很轻的博客系统，它不用依赖数据库，支持Markdown和输出静态页面，还可以一键部署到Github上。因此使用它和Github pages可以非常简单的搭建一个个人博客。

在本地搭建Hexo环境（在Linux/Mac OS）只需要以下几个简单步骤:

  • 首先需要安装node.js(此步略过)
  • 安装Hexo
  • 使用Hexo创建个人博客
  • 配置文件

安装 Hexo

安装Hexo只需要几一个简单的命令，使用npm包管理器可以很简单的从命令行安装。 -g的含义是使用Globle来安装，这样Hexo就被自动配置到环境变量了。

创建个人博客

安装完Hexo之后，使用以下指令可以很简单的创建博客

配置博客

使用命令进入到博客的根目录，使用npm install即可把博客所需要的包安装到目录内

创建第一篇博文

使用 “hexo n 博文名” 即可创建博文，创建的博文会放在./source/_posts内。

启动Hexo

最后使用以下指令即可以启动博客服务器，默认端口为4000，可以在配置文件“_config.yml”中更改，启动博客后，访问“localhost:4444”即可看到自己架设的博客。在博客运行中，使用Ctrl+C可以关闭博客页面

也可以使用$ hexo s &以下命令让hexo在后台运行,然后用kill将它结束

以上，一个简单的博客就在本地架设好了。
