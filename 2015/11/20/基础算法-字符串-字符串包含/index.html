    基础算法-字符串-字符串包含

    One Punch CAT
    2015-11-20
    leetcode, lintcode, 字符串, 算法

    字符串包含问题算是非常基础的问题，最典型的是两类问题

    1. 主串中是否包含子串
    2. 字符串A是否包含字符串B所有的字符

    里收录几个不同类型的字符串包含题。

    Compare Strings - lintcode

    问题：

    比较串A和串B，判断串A是否包含串B中所有的字符。串AB都仅包含大写字符。
    Compare two strings A and B, determine whether A contains all of the characters in B. The characters in string A and B are all Upper Case letters.

    例子：

    • A = “ABCD”, B = “ACD”, return true.
    • A = “ABCD”, B = “AABC”, return false. 这里注意，lintcode中的这道题目，在 A=”A” B=”AA”这种情况下返回的是false。

    分析：

    1. 对于是否出现，这种查询类的问题，标准的做法是使用哈希表。
    2. 如果A=”A” B=”AA”的这种情况返回的是true的话，那么仅判断字符是否存在于串A就行了。对于判断是否存在，这类只有「0」和「1」两种情况的问题，可以使用BitSet或者一个整数和位操作来减少空间的使用量。
    3. 但是考虑到本例中，都是大写字母，最多出现26个字符。对于这种key的范围和数量是已知的问题，可以使用一个数组来记录，操作更加方便。这里使用int[26]来记录。

    strStr – lintcode

    问题

    判断子串是否存在与主串中，若存在，输出主串中子串的座标，若不存在，输出-1。
    For a given source string and a target string, you should output the first index(from 0) of target string in source string.If target does not exist in source, just return -1.

    例子

    • If source = “source” and target = “target”, return -1.
    • If source = “abcdabcdefg” and target = “bcd”, return 1.

    分析

    1. 最普通的做法就是从主串头开始一个一个字符匹配，遇到不匹配，回溯主串，复杂度O(n^2)。
    2. 另外可以用KMP算法，不回溯主串，直接通过覆盖函数来找到下一个匹配点，复杂度O(n+m)。对于KMP不熟悉的同学，可以参照我翻译的一个视频汪都能听懂的KMP字符串匹配算法

    Group Anagrams

    问题

    输入一组字符串，将所有互为anagram的字符串打包成一组输出
    Given an array of strings, group anagrams together.

    例子

    • [“eat”, “tea”, “tan”, “ate”, “nat”, “bat”] => [[“ate”, “eat”,”tea”],[“nat”,”tan”],[“bat”]]

    分析

    1. Anagrams的单词在排序之后一定都是一样的，所以可以以排序后的单词为key, value一个与这个key Anagram单词的List
