无向图检测环 - Graph Valid Tree

One Punch CAT
2015-11-28
leetcode, 图, 拓扑排序, 环检测, 算法

问题

给出n个节点，节点的值的范围是[0,n-1]，以及一个边的List，判断这个图是不是一棵树。
Given n nodes labeled from 0 to n - 1 and a list of undirected edges (each edge is a pair of nodes), write a function to check whether these edges make up a valid tree.

例子

  • n = 5, 边 = [[0, 1], [0, 2], [0, 3], [1, 4]] -> true.

  • n = 5, 边 = [[0, 1], [1, 2], [2, 3], [1, 3], [1, 4]] -> false.

分析

  1. 首先这道题是判断图是不是一棵树，因此要从树的性质出发：

    • 树是一种无向图
    • 树任意两个顶点间存在唯一的路径，也就是图中没有环
    • 树只有一个根节点，否则就是森林

  2. 所以这道题目的本质就是判断一个无向图中是否只有一个根节点，并且是否存在环。

  3. 第一步是构建无向图，我使用的是邻接表的形式，存下与每个节点相邻的节点list。
  4. 检测环有两种方法：
    • 使用拓扑排序，每次访问图中入度为1的节点，依次取出，将这个节点相邻的节点的入度减1，再找到图中所有入度为1的节点，注意是从整个图而不是被访问节点的相邻节点找，直到graph中没有节点为止。如果graph中找不到入度为1的节点，说明剩下的节点是一个环。一开始判断一下入度为1的节点是否只有一个，如果不是，则说明是森林。
    • 创建一个visit数组记录每一个节点是否已经被访问，因为是无向图，因此从图中任意节点出发，遍历这个图，判断与每一个节点连通的所有节点中，除了这个节点的上一个节点，是否有被访问过的节点，如果有，则说明有环。最后再判断一下图中所有的点是否都已经被访问，如果没有，则说明是森林不是树。
