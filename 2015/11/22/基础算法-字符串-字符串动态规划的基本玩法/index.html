    基础算法-动态规划-动态规划的基本玩法

    One Punch CAT
    2015-11-22
    leetcode, lintcode, 字符串, 数组, 算法

    动态规划，我一开始听到这个算法的名字，觉得这是一个很难的算法。对的，动态规划难的题目确实很难，然而其中一些基础的玩法还是很容易理解的。动态规划最重要的思路就是，把大问题分解成许多同样解法的小问题，从size最小的情况开始考虑，一步一步记录结果，直到size N。 因此对于动态规划，找到递推表达式，就是胜利。对于一些基础的问题来说，这个递推表达式是很容易得到的。所以来看一下一些最容易的动态规划题。

    Longest Increasing Subsequence - leetcode

    问题：

    说到动态规划，怎么能不说大名鼎鼎的LIS问题？ 问题的大致意思就是从一个乱序的数组中，找到最长的递增序列，并返回这个序列的长度。要注意，sequence序列是可以不连续的。
    Given an unsorted array of integers, find the length of longest increasing subsequence.

    例子：

    [10, 9, 2, 5, 3, 7, 101, 18] => 4 LIS是 [2 ,3 ,7 ,101]

    分析：

    • 创建一个和数组A大小一样的dp数组dp，对于每一个下标i, dp[i]用来记录包含A[i]这个数字的LIS的结果。
    • 从下标0开始考虑，如果数组只有[10]，对于[10]来说，LIS是1。所以dp[0] = 1。
    • 对于下标n，LIS是的计算方式是，遍历他之前所有下标的dp值dp[i]，如果A[n] > A[i], 那么计算dp[i] + 1。 取所有计算值中的最大值。如果所有的A[i]都大于A[n]，那么取1。
    • 最后返回dp这个数组中的最大值。
    • 这个算法的复杂读是O(n^2)

    Longest Common Substring - lintcode

    问题：

    给定两个字符串, 找到字符串A和字符串B中最长的公共子字符串。注意这道题是substring, 因此是需要连续的。

    例子：

    对于 A = “ABCD”, B = “CBCE”, 返回2。Longest Common Substring是BC

    分析：

    • 这道题目因为有两个input argument, 所以我们需要一个二维dp数组来记录。
    • dp[i][j]表示以字符串A的第i个字符串B的第j个字符为匹配的最后一个字符的连续字符串长度
    • 如果A[i]=B[j], 那么dp[i][j] = dp[i-1][j-1]+1。即为A的i-1个字符与B的j-1个字符的连续匹配情况
    • 如果A[i]!= B[j], 那么dp[i][j] = 0。即从A[i]和B[j]开始，连续的匹配被打破。
    • 如果A[i]!= B[j], 如果取dp[i-1][j]和dp[i][j-1]中的最大值，那么就是Longest Common Sequence的解法。
